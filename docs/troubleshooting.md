# Troubleshooting

## Launch & workspace errors

- **"Missing required option --workspace."** — every launch needs `--workspace <absolute-path>` (both transports).
- **"--workspace must be an absolute path"** — relative paths are rejected; pass the full path (in mcp.json use `${workspaceFolder}` in VS Code).
- **"Workspace root not found / not a directory / not readable"** — check the path exists, is a directory, and your user can read it.
- **"--port is required with --http"** — HTTP mode always needs an explicit port.
- **"port `<n>` is already in use"** — another process (possibly a previous server instance) owns the port; stop it or pick a different `--port`.

## Install & asset resolution

- **"Grammar assets not found. Expected directory: …"** — the install is incomplete:
  - registry/npx install: clear the cache (`npm cache clean --force`) and reinstall — the tarball ships `assets/grammars/`;
  - git/GitHub install: ensure install ran with devDependencies (default for git deps) so the `prepare` build could fetch grammars;
  - source checkout: run `node scripts/fetch-grammars.mjs` (or `npm install`).
- **GitHub install fails building** — Node ≥ 20 required; retry with `npm install -g github:AnhNguyenDaenet/WorkplaceWorkTree --foreground-scripts` to see the build output.
- **Different `--version` across machines** — channels can lag one another (e.g. `latest` Docker tag vs npm); pin exact versions when it matters.

## Scanning

- **Huge repo, slow scan** — add `excludePatterns` (e.g. `["third_party/**"]`); verify what was skipped in the document's "Exclusions applied" header.
- **A language shows "fallback (imports only)"** — deep analysis covers C#, TS/JS, Python, Java, Go, Rust; other languages get file-level imports only.
- **Maps look stale** — check the `Generated by … on <timestamp>` header; run `update_maps` (or `{"force": true}` for a full rebuild).
- **Report says `queued for <n> ms`** — another tool call was running; executions are serialized by design. Purely informational.

## Guidance & init

- **Duplicate managed-section error** — your copilot-instructions.md contains more than one `<!-- BEGIN workspace-map-mcp -->` block; delete the extras and re-run `install_guidance`.
- **`init` aborts with a JSONC error** — your `.vscode/mcp.json` has comments/trailing commas; strict JSON is required. Remove them or paste an entry manually from [transports.md](./transports.md). The file is never modified on abort.
- **`init` ran but the client doesn't see the server** — reload the MCP client (VS Code: "MCP: List Servers" → start `workspace-map`); for `--transport http` entries, start the server process first.

## Docker

- **Maps not appearing on the host** — the project must be bind-mounted (`-v <project>:/workspace`) and `--workspace /workspace` passed; without `-v` the maps die with the container.
- **Permission errors on Linux hosts** — add `--user $(id -u):$(id -g)` to `docker run` so mount writes use your uid.
- **HTTP container unreachable** — inside containers use `--host 0.0.0.0` and publish the port with `-p <port>:<port>`; loopback binding is unreachable from the host.

## Development

```powershell
npm install         # deps + WASM grammars
npm test            # vitest: contract, unit, integration suites
RUN_PERF=1 npm test # opt-in 10k-file performance benchmark
npm run lint        # eslint
npm run build       # tsc -> dist/
```

Generated artifacts land in the *target* workspace's `.codemap/` — never in this repository.
