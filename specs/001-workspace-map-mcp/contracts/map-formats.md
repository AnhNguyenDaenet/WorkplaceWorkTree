# Contract: Generated Map Document Formats

**Feature**: 001-workspace-map-mcp | **Date**: 2026-07-07
**Consumers**: AI assistants (primary), humans (secondary). These formats are a stable contract — agent skills and copilot-instructions guidance reference them, so changes require a version bump in the metadata header.

## Common metadata header (both documents, FR-008)

Every generated document begins with:

```markdown
# <Document Title>

> Generated by workspace-map-mcp v<semver> on <ISO-8601 timestamp>
> Workspace: <workspace root folder name>
> Format version: 1

| Metric | Value |
|---|---|
| ... summary counts ... | ... |

## Exclusions applied
- Built-in defaults: `.git`, `node_modules`, ...
- From .gitignore: <count> rules across <n> files
- User patterns: <list or "none">
```

## `.codemap/structure.md` (FR-001)

```markdown
# Workspace Structure Map
> ...metadata header...

## Tree

```text
<workspace-name>/
├── src/
│   ├── app/
│   │   ├── main.cs                  # src/app/main.cs
│   │   └── util.cs                  # src/app/util.cs
│   └── web/ → (symlink, recorded once)
└── README.md                        # README.md
```
```

**Rules**:

- Folders before files, both alphabetical (case-insensitive), guaranteeing deterministic diffs.
- Every file line ends with a `# <workspace-relative-path>` comment so an AI can copy the exact relative path without reconstructing it from tree indentation (FR-001 "relative path readable per entry").
- Symlinked/junction directories appear once with a `→ (symlink...)` marker and are not expanded again.
- Paths always use `/` separators regardless of OS.

**Partitioned form (FR-015)**: when the tree exceeds `maxDocLines`, `structure.md` keeps the metadata header + top-level-only tree + a partition index table (`| Folder | Document | Files |`) linking to `structure/<top-level-folder>.md`, each of which repeats the header and contains that folder's full subtree.

## `.codemap/relations.md` (FR-003, FR-004, FR-011, FR-013)

```markdown
# Code Relation Map
> ...metadata header...

## Language coverage

| Language | Files | Analysis tier |
|---|---|---|
| C# | 120 | deep |
| Markdown | 14 | none |
| PowerShell | 3 | fallback (imports only) |

## Type index

| Type | Kind | Qualifier | Defined in |
|---|---|---|---|
| `OrderService` | class | `Contoso.Orders` | src/orders/OrderService.cs |
| `OrderService` | class | `Contoso.Billing` | src/billing/OrderService.cs |

> ⚠ Name collisions are listed with full qualifiers — always confirm the qualifier before navigating.

## Types

### `Contoso.Orders.OrderService` (class, C#) — src/orders/OrderService.cs
- **Inherits**: `ServiceBase` → `Contoso.Core.ServiceBase` (src/core/ServiceBase.cs)
- **Implements**: `IOrderService` → `Contoso.Orders.IOrderService` (src/orders/IOrderService.cs)
- **Calls** *(syntactic, best-effort)*: `Repository.Save`, `Validator.Check`

## File dependencies

| From | To | Via |
|---|---|---|
| src/orders/OrderService.cs | src/core/ServiceBase.cs | `using Contoso.Core` |
| src/web/app.ts | (external) | `import express` |

## Reduced analysis
- src/legacy/broken.cs — parse errors, file-level analysis only
- scripts/deploy.ps1 — fallback tier (imports only)
```

**Rules**:

- Type sections sorted by qualified name; every file reference is workspace-relative (FR-004) and must exist in structure.md's tree (cross-document consistency).
- Unresolved relation targets (external packages) rendered as `(external)` — never as broken workspace paths.
- Call data always labeled *syntactic, best-effort* (v1 confidence level).
- Docs-only workspace: document states "No analyzable source code found" under Language coverage.

**Partitioned form**: same scheme — root doc keeps header, language coverage, full type index (the index is the navigation-critical piece and stays whole), and links to `relations/<top-level-folder>.md` for per-folder type detail sections.

## Managed copilot-instructions section (FR-010)

```markdown
<!-- BEGIN workspace-map-mcp -->
## Workspace maps (generated)

- `.codemap/structure.md` — full folder/file tree; read this to resolve any relative path instead of listing directories.
- `.codemap/relations.md` — type→file index, inheritance, imports, calls; read this to jump straight to a type's defining file instead of searching.
- If either file is missing or looks stale (see its generation timestamp), call the `update_maps` tool on the `workspace-map-mcp` MCP server.
<!-- END workspace-map-mcp -->
```

**Rules**: exactly one marker pair; content between markers is wholly owned by the tool; everything outside is untouchable user content.
